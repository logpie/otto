### Key observations

- **Spec gen timing is accurate**: 68s for $0.26 — proportional (thread-based parallelism working)
- **Spec gen always ready**: In all observed runs, spec gen finished before QA needed it
- **Typical cost breakdown**: coding ~55%, QA ~25%, spec gen ~18%, prepare ~2%
- **Multi-attempt tasks**: 3/9 tasks needed 2 attempts (QA caught real bugs: missing null guards, incorrect thresholds, test updates needed)
- **No browser testing observed**: QA consistently used code inspection + test execution, even for visual specs. Prompt instructs browser for ◈ items but agent prefers code reading. Known gap — needs programmatic enforcement.
- **◈ markers**: Spec agent generates them for visual items. Fuzzy matching in QA verdict works but not 100% reliable across attempts.

### Comparison vs bare CC

Not formally A/B tested in this session, but based on prior v4.5 testing:
- Otto coding agent IS bare CC (same SDK preset, no custom system prompt)
- Otto adds ~2-3 min overhead (spec gen + QA + verify), catching real bugs in 33% of tasks
- Total cost ~$2/task avg including QA
