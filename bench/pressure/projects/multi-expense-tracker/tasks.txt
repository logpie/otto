Build the data and validation layer for an expense tracker in Node.js. Create models: Expense (id, amount, currency, category, description, date, tags[]), Budget (id, category, monthly_limit, currency). Create ExpenseStore with in-memory storage and CRUD operations. Validation: amount must be positive number, currency must be 3-letter ISO code, date must be valid ISO string, category required. Include tests for validation edge cases and all CRUD operations.
Build the analytics engine on top of the data layer. Create analytics.js with: totalByCategory(month) returning {category: total}, totalByMonth(year) returning {month: total}, budgetStatus(month) returning [{category, spent, limit, remaining, overBudget}], topExpenses(n, {month, category} filters), averageDailySpend(month), spendingTrend(months) returning array of monthly totals for trend analysis. Handle currency conversion: accept an exchangeRates object mapping currency pairs to rates, normalize all amounts to a base currency before aggregation. Include tests with multi-currency data and budget threshold scenarios.
Build a REST API with Express on top of the analytics engine. Endpoints: POST /expenses, GET /expenses (with query filters: category, month, minAmount, maxAmount, page, limit), GET /expenses/:id, DELETE /expenses/:id, POST /budgets, GET /analytics/category-totals?month=YYYY-MM, GET /analytics/budget-status?month=YYYY-MM, GET /analytics/trends?months=6. Request validation middleware. Error handling middleware with proper status codes. Tests with supertest for all endpoints including validation errors and analytics correctness through the API.
