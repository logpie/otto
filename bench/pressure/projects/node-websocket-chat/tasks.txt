Build a WebSocket chat server using the ws library. Requirements: clients connect and send {type, payload} JSON messages. Supported types: 'join' (join a room with username), 'message' (send to current room), 'leave' (leave room), 'list_rooms' (get active rooms), 'list_users' (get users in current room). Server broadcasts messages to room members only. Handle: duplicate usernames in same room (reject with error), client disconnect (clean up from room, notify others), empty room cleanup (remove room when last user leaves), message history (last 50 messages per room, sent to client on join). Include heartbeat: server pings every 30s, client must pong within 10s or gets disconnected. Tests: use ws client library to test join/leave/message flow, room isolation, duplicate username rejection, disconnect cleanup, message history delivery. Use ws, jest.
