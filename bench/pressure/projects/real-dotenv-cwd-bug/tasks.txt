The python-dotenv CLI crashes if the current working directory has been deleted before importing dotenv. When a user runs a dotenv CLI command from a directory that no longer exists, os.getcwd() raises FileNotFoundError at module import time because the default --file option evaluates os.path.join(os.getcwd(), '.env') eagerly. Fix the CLI so that it gracefully handles a missing current working directory instead of crashing. The fix should be in src/dotenv/cli.py. Run tests with: pytest
