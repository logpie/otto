Build a CLI task manager with SQLite persistence. Requirements: commands — 'add <title> [--priority high|medium|low] [--due YYYY-MM-DD]', 'list [--status pending|done|overdue] [--sort priority|due|created]', 'done <id>', 'delete <id>', 'search <query>' (searches title and notes). Tasks have: id, title, notes (optional), priority, status, due_date, created_at, completed_at. Use argparse with subcommands. SQLite database stored at ~/.taskmanager/tasks.db (configurable via TASKMANAGER_DB env var). 'list' shows overdue tasks highlighted. 'search' uses SQL LIKE for fuzzy matching. Include tests using a temp database (not the real one). Test each subcommand including edge cases: duplicate adds, deleting nonexistent tasks, listing with filters, overdue detection. Use only stdlib plus pytest.
