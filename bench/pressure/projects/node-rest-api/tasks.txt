Build a REST API for a bookmark manager using Express. Requirements: CRUD endpoints — POST /bookmarks (create with url, title, tags[]), GET /bookmarks (list with ?tag=X&search=Q&page=1&limit=20 filtering), GET /bookmarks/:id, PUT /bookmarks/:id, DELETE /bookmarks/:id. Validation middleware: url must be valid URL format, title required and max 200 chars, tags must be array of strings. Store in-memory (array). Include: request ID middleware that adds X-Request-Id header, error handling middleware that returns {error, requestId} JSON, input sanitization against XSS in title/tags. Rate limiting: max 100 requests per IP per minute using in-memory counter. Tests with supertest covering: all CRUD ops, validation errors, pagination, tag filtering, search, rate limit triggering, request ID propagation. Use express, jest, supertest.
