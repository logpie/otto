Add subcommand alias support to citty. Allow defining aliases on subcommands via an 'alias' field in the command meta definition, accepting string | string[]. When a user types an alias (e.g. 'i' for 'install'), it should resolve to the corresponding subcommand. Direct key matches must take priority over alias matches. Aliases should also appear in --help output (e.g. 'install, i, add'). Must work with nested subcommands. Add tests that verify: (1) basic alias resolution, (2) array of aliases, (3) direct key match takes priority over alias, (4) alias appears in help text, (5) nested subcommand aliases work, (6) error for unrecognized command/alias. Run tests with: npx vitest run
