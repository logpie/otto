Build the data layer for a blog engine in Python. Create models.py with Post (id, title, slug, content, author, tags[], status=draft|published, created_at, updated_at) and Comment (id, post_id, author, content, created_at) classes. Create store.py with an in-memory PostStore that supports: create, get_by_id, get_by_slug, list(status=, tag=, author=, page=, limit=), update, delete. Slug must be auto-generated from title (lowercase, hyphens, no special chars) and unique. Include full tests for all CRUD operations, slug generation, filtering, and pagination edge cases.
Build the business logic layer on top of the data layer. Create blog.py with BlogService that uses PostStore. Features: publish(post_id) changes status and sets updated_at, add_comment(post_id, author, content) with validation (no empty content, post must be published), get_post_with_comments(slug) returns post + comments, get_feed(page, limit) returns published posts newest first, search(query) searches title and content (case-insensitive substring match), get_tag_cloud() returns {tag: count} for published posts. Include tests that exercise the full stack: create post, publish, comment, search, feed ordering.
Build a CLI interface using argparse on top of BlogService. Commands: 'new <title> --author <name> --tags tag1,tag2', 'publish <slug>', 'comment <slug> --author <name> --content <text>', 'list [--status draft|published] [--tag X] [--page N]', 'show <slug>' (displays post + comments formatted), 'search <query>', 'tags' (shows tag cloud). CLI reads/writes a JSON file for persistence (serialize/deserialize the in-memory store). Include tests that invoke the CLI via subprocess and verify output format and JSON file state.
