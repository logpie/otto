Build an in-memory task queue library with priority scheduling. Requirements: TaskQueue class with enqueue(task, {priority, maxRetries, delayMs}), process(concurrency) that runs tasks in parallel up to concurrency limit, pause()/resume(), drain() that resolves when all tasks complete. Priority levels: high (0), medium (1), low (2) — higher priority dequeued first. Retry logic: on failure retry up to maxRetries with exponential backoff (100ms, 200ms, 400ms...). Dead letter queue: tasks exceeding maxRetries go to a DLQ accessible via getDLQ(). Events: emit 'completed', 'failed', 'retrying', 'dlq' events with task info. Include a RateLimitedQueue subclass that limits throughput to N tasks per second. Tests: priority ordering, concurrency limit respected (verify no more than N tasks run simultaneously), retry with backoff timing, DLQ population, pause/resume behavior, rate limiting accuracy. Use jest, no external queue libraries.
