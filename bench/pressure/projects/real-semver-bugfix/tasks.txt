Fix the prerelease identifier parsing bug in node-semver. The bug: prerelease identifiers that start with digits but contain letters are truncated. For example, '1.0.0-alpha.1ab' is coerced to '1.0.0-alpha.1', and '1.0.0-alpha.12ab' becomes '1.0.0-alpha.12'. The issue is in the regex for PRERELEASEIDENTIFIER where NUMERICIDENTIFIER is matched before NONNUMERICIDENTIFIER — since numeric matches greedily on leading digits, a mixed identifier like '1ab' matches as '1' instead of '1ab'. Fix the regex ordering so non-numeric identifiers are tried first, and add tests that verify: (1) '1.0.0-alpha.1ab' parses correctly with prerelease ['alpha', '1ab'], (2) '1.0.0-beta.12ab34' preserves the full identifier, (3) pure numeric identifiers like '1.0.0-alpha.1' still work, (4) coerce() handles these cases too. Run tests with: npm test
