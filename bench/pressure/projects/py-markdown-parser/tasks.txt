Build a Markdown-to-HTML converter library in Python. Requirements: parse a subset of Markdown — headings (h1-h6), paragraphs, bold (**text**), italic (*text*), inline code (`code`), code blocks (```lang), unordered lists (- item, nested), ordered lists (1. item), links ([text](url)), images (![alt](src)), blockquotes (> text, nested), horizontal rules (---). Architecture: Lexer that tokenizes input into block-level tokens, then Parser that converts tokens to an AST, then Renderer that walks the AST and produces HTML. Must handle edge cases: empty input, unclosed formatting, nested formatting (**bold *and italic***), consecutive blank lines, mixed list types. Include tests for each component (lexer, parser, renderer) and end-to-end tests with realistic markdown documents. Use only stdlib plus pytest.
