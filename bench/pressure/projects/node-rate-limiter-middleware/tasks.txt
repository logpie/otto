Build a configurable rate limiting middleware library for Express. Requirements: three algorithms — (1) Fixed Window: count requests per time window, reject when limit exceeded, (2) Sliding Window Log: track timestamps of recent requests for smoother limiting, (3) Token Bucket: refill tokens at steady rate, consume on request. Factory function createRateLimiter({algorithm, limit, windowMs, keyGenerator}) returns Express middleware. keyGenerator extracts the rate limit key from request (default: req.ip). Response when limited: 429 status with Retry-After header and {error, retryAfter} JSON body. Include X-RateLimit-Limit, X-RateLimit-Remaining, X-RateLimit-Reset headers on all responses. Support skipList: array of keys that bypass limiting. Support custom onLimitReached callback. Tests with supertest: each algorithm enforces limits correctly, Retry-After header accuracy, skip list works, headers present on normal responses, different keys get independent limits, window reset behavior. Use express, jest, supertest.
