The humanize.precisedelta() function has rounding issues with certain timedelta values. For example, precisedelta(dt.timedelta(seconds=3.4), minimum_unit="seconds", format="%0.1f") should return "3.4 seconds" but returns incorrect results due to improper rounding at the minimum_unit boundary. The _quotient_and_remainder helper divides value/divisor and returns a float when at minimum_unit, but doesn't apply the format string's rounding, causing cascading inaccuracies. Fix the precisedelta rounding so that when the minimum_unit is reached, the format string is used to properly round the quotient value. Run tests with: pytest
