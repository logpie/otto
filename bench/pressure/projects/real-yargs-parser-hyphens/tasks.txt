The yargs-parser incorrectly handles options that end with 3 or more hyphens (like "--foo---"). The regex /---+(=|$)/ that detects invalid multi-hyphen arguments is missing the start anchor ^, so it matches options like "--foo---" where "---" appears at the end (matching the trailing "---" portion). This causes these arguments to be treated as invalid positionals instead of being parsed as the option "--foo---". Fix the regex in lib/yargs-parser.ts to anchor it at the start with ^, so it only matches arguments that START with 3+ hyphens (like "---" or "----"), not arguments that merely contain "---" somewhere in the middle or end.
