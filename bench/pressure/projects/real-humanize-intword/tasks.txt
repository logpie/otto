The humanize.intword() function has two bugs: (1) it uses incorrect plural forms — for example, intword(1234567, "%.0f") returns "1 millions" instead of "1 million" because it uses math.ceil(chopped) for the ngettext count, and (2) boundary values are handled incorrectly — intword(999500, "%.0f") should return "1 million" (since 999500 rounds up to 1M with zero decimal places) but instead returns "999 thousand" or worse. Fix intword() to use proper plural logic (the formatted/rounded value determines singular vs plural) and handle boundary rounding correctly (when the formatted value rounds up to the next power, bump to the higher unit).
