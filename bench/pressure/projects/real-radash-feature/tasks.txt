Add an inRange(number, start, end?) utility function to the radash number module. Equivalent to lodash's _.inRange — checks if the given number is between two bounds. Start is inclusive, end is exclusive. If only two args provided (number, end), range is [0, end). Handles the case where start > end by swapping the bounds. Returns false for non-number inputs (null, undefined, NaN). Export from the main module. Add comprehensive tests: basic range check, two-arg form, inverted range, boundary values (inclusive start, exclusive end), non-number inputs, negative ranges, zero edge cases.
