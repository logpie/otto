Fix the box_dots get() regression in python-box v7.3.1. The bug: Box.get() with box_dots=True stopped returning the default value for missing nested keys. For example, Box({'foo': {'bar': 1}}, box_dots=True).get('foo.grab.plum', 4) should return 4 (since 'grab' is not a key of 'foo'), but instead raises an error. This was a regression introduced while fixing issue #275. Fix the get() method to properly return the default value when any segment of a dotted key path is missing, and add tests that verify: (1) get with existing dotted path returns the value, (2) get with missing intermediate key returns default, (3) get with missing final key returns default, (4) get with no default raises/returns None appropriately, (5) existing box_dots behavior for set/access is unchanged. Run tests with: pytest
