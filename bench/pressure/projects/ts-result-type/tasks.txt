Build a Result/Option monad library in TypeScript with full type inference. Requirements: Result<T, E> type with Ok(value) and Err(error) constructors. Methods: map(fn), mapErr(fn), flatMap(fn) (also called andThen), unwrap() (throws on Err), unwrapOr(default), match({ok, err}) for pattern matching, isOk(), isErr(). Option<T> type with Some(value) and None. Methods: map, flatMap, unwrap, unwrapOr, match({some, none}), isSome, isNone. Conversion: Result.ok() returns Option, Option.okOr(err) returns Result. Utility: Result.all([results]) returns Result<T[], E> (first error), Result.any([results]) returns first Ok. Async support: ResultAsync<T, E> wrapping Promise<Result<T, E>> with map/flatMap that chain async operations. fromPromise(promise, errorMapper) converts Promise to ResultAsync. All types must be fully inferred — no explicit type annotations needed by callers. Tests: every method on Result and Option, chaining, all/any combinators, async operations, type inference compile checks. Use typescript, jest, ts-jest.
