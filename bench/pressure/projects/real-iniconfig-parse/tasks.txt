The iniconfig library currently does not handle inline comments properly when parsing INI files. Values like "key = value # this is a comment" retain the "# this is a comment" part in the parsed value. Add an IniConfig.parse() classmethod that accepts a data string and a strip_inline_comments parameter. When strip_inline_comments=True, inline comments (starting with # or ;) should be stripped from values. Also add a strip_section_whitespace parameter that strips whitespace from section names and key names. The parse() method should be a classmethod on IniConfig that returns a new instance. Run tests with: pytest
