Build a streaming file processor for JSONL files. Requirements: FileProcessor class that reads JSONL files line-by-line using Node streams (not loading entire file into memory). Pipeline stages: (1) Parser — parse each line as JSON, collect parse errors with line numbers, (2) Validator — validate against a user-provided schema object {field: 'string'|'number'|'email', required: bool}, reject invalid records, (3) Transformer — apply user-provided transform function to each valid record, (4) Writer — write results to output JSONL file. Must handle: empty lines (skip), parse errors (log and continue), very large files (streaming, bounded memory), backpressure (pause reading when writing is slow). Include a summary report after processing: total lines, parsed, valid, invalid, errors with line numbers. Tests: process fixtures/valid.jsonl and fixtures/mixed.jsonl, verify streaming behavior with mock slow writer, test schema validation, test transform pipeline, verify error report accuracy. Use jest, Node streams API only.
