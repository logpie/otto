When using pathspec with an empty list of patterns, certain backends (like the re2 and hyperscan backends) raise a ValueError saying "patterns cannot be empty" instead of gracefully returning no matches. The default regex backend handles this case fine, but the re2 and hyperscan backends explicitly reject empty pattern lists. Fix the backends so that empty pattern lists are handled gracefully — the backends should initialize without error and return no matches when given empty patterns, rather than raising ValueError. The fix should be in pathspec/_backends/re2/pathspec.py and pathspec/_backends/hyperscan/pathspec.py. Run tests with: pytest
