Build a URL shortener service with Express. Acceptance criteria: (1) POST /shorten accepts {url, customAlias?, expiresIn?} and returns {shortUrl, alias, expiresAt}, (2) GET /:alias redirects to original URL with 301, (3) GET /:alias/stats returns {alias, originalUrl, clicks, createdAt, lastClickedAt, clicksByDay, topReferrers, topCountries}, (4) custom aliases must be 3-20 alphanumeric chars, (5) auto-generated aliases are 7 chars base62, (6) expired links return 410 Gone, (7) duplicate URL returns existing short link, (8) rate limit: 10 shortens per IP per hour, (9) click tracking records: timestamp, referrer header, user-agent, (10) GET /api/top returns top 10 most clicked links, (11) GET /api/recent returns 20 most recently created links, (12) input validation: reject non-http(s) URLs, reject URLs longer than 2048 chars, (13) store everything in-memory with Maps, (14) CORS headers on all responses, (15) health check at GET /health returning uptime and total links count, (16) all error responses use {error, code} JSON format with appropriate HTTP status. Include tests with supertest covering every acceptance criterion. Use express, jest, supertest.
