Add a 'persist_empty' feature to TinyDB tables. Currently, calling db.table('name') does not create any storage entry until data is inserted. Add a persist_empty parameter to Table (default False). When True, the table is immediately written to storage even when no documents are inserted — similar to SQL CREATE TABLE. This is useful when you want to ensure a table exists in storage before inserting data, or to persist the table structure for other processes to discover. Include tests that verify: (1) table with persist_empty=True appears in storage immediately, (2) table with persist_empty=False (default) does NOT appear until insert, (3) clearing a persist_empty table keeps the empty entry in storage, (4) existing behavior is unchanged when persist_empty is not set. Run tests with: pytest
