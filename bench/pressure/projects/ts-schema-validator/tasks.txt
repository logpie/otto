Build a runtime schema validation library in TypeScript (like a mini zod). Requirements: chainable API — s.string(), s.number(), s.boolean(), s.array(schema), s.object({key: schema}), s.union([schema1, schema2]). Each schema has .parse(value) that returns typed value or throws ValidationError, and .safeParse(value) that returns {success, data} | {success, error}. Modifiers: .optional(), .nullable(), .default(val), string-specific: .min(n), .max(n), .email(), .regex(pattern), number-specific: .min(n), .max(n), .int(). ValidationError must include path to the failing field (e.g. 'address.zip' for nested objects). TypeScript types must infer correctly: s.object({name: s.string(), age: s.number()}).parse(x) should return {name: string, age: number}. Tests: every type, every modifier, nested objects, arrays of objects, union discrimination, error paths, type inference (compile-time tests using type assertions). Use typescript, jest, ts-jest.
