The file-contents-sorter hook adds a blank line to empty files. When an empty file is processed by sort_file_contents(), the code unconditionally appends a newline to the sorted output (b'\n'.join(after) + b'\n'), which means an empty file gets transformed to contain just a newline character. This causes the hook to report that the file was modified when it shouldn't have been. Fix the file_contents_sorter.py so that empty files remain empty after sorting — only append the trailing newline if there is actual content. Run tests with: python -m pytest
