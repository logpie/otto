### Observations
- **QA time is significant** — often 40-50% of total time. The QA agent reads tests, runs targeted checks, and verifies spec items.
- **Spec generation** (otto add) takes 30-90s per task. Multi-task projects spend significant time in this phase.
- **Simple bugfix projects** are fastest (~4 min) because the scope is well-defined.
- **TypeScript projects** tend to take longer — type inference and generics require more coding agent iterations.

## Codex Review Findings (future improvements)

Codex reviewed the project set and identified:
- **False-pass risk**: Otto grades its own homework (writes tests, then passes them)
- **Open-book bugfix**: Bug comments in code tell agent exactly where to look
- **Underspecified specs**: Multiple incompatible implementations can "pass"
- **Coverage gaps**: No frontend, database, monorepo, CI/Docker, or upgrade tasks
- **--forceExit masks leaks**: Node test starters hide resource leak bugs

Full details in `/tmp/otto-pressure-bugs.md` under FUTURE IMPROVEMENTS.
