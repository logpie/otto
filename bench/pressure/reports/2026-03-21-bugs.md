## BUG-1: No baseline test check before coding [FIXED]

**Project:** real-semver-bugfix
**Severity:** major
**Status:** Fixed in commit 627bdc0 (pressure-test branch)
**Symptom:** Existing tests fail every attempt ($13.81 burned). The error was `Cannot find module 'map.js'` — a project setup issue, not a code issue. The agent's regex fix was correct but could never be verified.
**Root cause:** Otto runs existing tests only AFTER coding. If the test suite is broken before any code changes, otto can't distinguish "your code broke tests" from "the project's test infrastructure was already broken."
**Fix:** Added baseline check in prepare phase — runs existing tests on unmodified code. If they fail, returns BASELINE_FAIL immediately.

## FUTURE IMPROVEMENTS (from Codex review + pressure test observations)

### High Priority

**IMP-1: False-pass risk — otto grades its own homework**
Greenfield projects ask otto to "include tests", then otto verifies against its own tests. No independent post-run check. Otto can write weak tests that pass with buggy code. Fix: add independent acceptance tests for greenfield projects, or a post-run verification step that checks behavior, not just test pass/fail.

**IMP-2: Bugfix projects are "open book"**
Bug comments in source code (`# BUG 1: ...`) tell the agent exactly where to look. Tests patch-following, not diagnosis. Fix: remove comments, or create "unlabeled" variants where the agent must find bugs from test failures alone.

### Medium Priority

**IMP-3: Specs are underspecified**
py-rate-limiter doesn't define refill precision, node-rest-api leaves XSS/pagination semantics vague. Multiple incompatible implementations can "pass." Fix: tighten specs with exact behavioral contracts.

**IMP-4: `--forceExit` masks resource leaks**
Node/TS starters use `npx jest --forceExit` which hides open handles. Unpinned npm packages cause version drift. Fix: pin versions in package.json, use `--detectOpenHandles` instead.

**IMP-5: edge-greenfield-complex is overloaded**
Combines project bootstrapping, file locking, query language, indexing, concurrency, and speedup assertion in one task from an empty repo. More endurance gauntlet than targeted probe. Fix: split into 2-3 focused tasks.

### Coverage Gaps (new project categories to add)

**IMP-6: Frontend/browser tasks** — React/Vue components, CSS regressions, state management
**IMP-7: Database/migration tasks** — schema changes, ORM, transactional correctness
**IMP-8: Monorepo/workspace tasks** — pnpm/turbo, shared packages, cross-package type drift
**IMP-9: CI/Docker/env-debugging tasks** — fix failing GitHub Action, Dockerfile, env-var wiring
**IMP-10: Upgrade/refactor tasks** — "bump library X one major version and adapt"
**IMP-11: Medium-sized existing-codebase changes** — most tasks are greenfield or narrowly localized
